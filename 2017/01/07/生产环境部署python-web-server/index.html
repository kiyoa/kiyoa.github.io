安装python

supervisor 依赖于python， 虽然系统6.5自带了python2.6 版本， 但是他的supervisor支持的最高版本太低， 不建议用系统自带python安装

1、python依赖组件安装

由于python很多库依赖于比较多的系统组件， 建议大家一定要进行下面的准备工作， 不然以后再进行一些库的使用时问题会比较多

2、编译安装

我这里安装的是python2.7版本， 由于python3以前编码问题比较严重， 建议在编译时配置命令的添加 –enable-unicode=ucs4 支持UTF-32，
在python3中加入–with-wide-unicode

3、安装pip、setuptools

4、创建一个虚拟python环境

既然有了python环境， 为什么还要创建一个虚拟python环境呢， 想想这种情况， 你有两个python web服务， 一个是依赖python2.7+Django1.2， 一个依赖python2.7 + Django1.6， Django这两个版本有很多不同， 如果你直接用你安装的python环境， 到时候， 你两个python web服务就不能同时启动了， 你有没有多的钱买服务器了。 这样就悲剧了， 可能你会说安装两个python2.7环境， 这 样也可以， 但是使用python 的虚拟环境是一个很容易的事， 为什么我们不做呢
